1  User Stories assignment 2  User Story “Create the database based on the relational model“ As a: developer I want to: create a database So that: the Someren application can use it for storing/retrieving information Checklist • An Azure database has been created (see manual SQL Server and Database creation manual); • SQL Server Management Studio has been installed; • All tables, columns and references (based on the relational model) has been created;  User Story “Prepare Visual Studio solution“ As a: developer I want to: prepare the Visual Studio solution  So that: the functionalities for the Someren application can be implemented Checklist • The Someren Visual Studio project (ASP.NET MVC) has been created; • The database connection string has been configured in the project; • A layout for all views of the Someren web application have been defined (including a top menu); • A Git repository has been created for the Visual Studio project;   
2  User Story “View all students“ (variant A) As a: user I want to: view all students So that: I know which students are participating the Someren camp Checklist • Test data (students) have been added to the database; • All students are displayed in a structured way, ordered by last name; • For each student the student number, first name, last name, telephone number and class is displayed;  User Story “Manage students” (variant A) As a: user I want to: manage students So that: I can view, add, change and delete students of the Someren camp Checklist • All students are displayed in a structured way, ordered by last name; • For each student the student number, first name, last name, telephone number and class is displayed; • A new student can be added; • An existing student can be changed (each field can be changed); • An existing student can be deleted; • All student (management) functions are accessible to the user by using links/buttons;    
3  User Story “View all lecturers” (variant B) As a: user I want to: view all lecturers So that: I know which lecturers are participating the Someren camp Checklist • Test data (lecturers) have been added to the database; • All lecturers are displayed in a structured way, ordered by last name; • For each lecturer the first name, last name, telephone number and age is displayed;  User Story “Manage lecturers” (variant B) As a: user I want to: manage lecturers So that: I can view, add, change and delete lecturers of the Someren camp Checklist • All lecturers are displayed in a structured way, ordered by last name; • For each lecturer the first name, last name, telephone number and age is displayed; • A new lecturer can be added; • An existing lecturer can be changed (each field can be changed); • An existing lecturer can be deleted; • All lecturer (management) functions are accessible to the user by using links/buttons;     
4  User Story “View all rooms” (variant C) As a: user I want to: view all rooms So that: I know which rooms are used by students and which rooms are used by lecturers Checklist • Test data (rooms) have been added to the database; • All rooms are displayed in a structured way, ordered by room number; • For each room the number, size and type (student/lecturer) is displayed;  User Story “Manage rooms” (variant C) As a: user I want to: manage rooms So that: I can view, add, change and delete rooms of the Someren camp Checklist • All rooms are displayed in a structured way, ordered by room number; • For each room the number, size and type (student/lecturer) is displayed; • A new room can be added; • An existing room can be changed (each field can be changed); • An existing room can be deleted; • All room (management) functions are accessible to the user by using links/buttons;
